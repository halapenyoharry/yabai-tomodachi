# yabai-tomodachi Ideas & Observations

## 🧠 Local AI Menu Bar Integration
Instead of just Claude Desktop, imagine a tiny local LLM in the menu bar that understands creative commands:
- "Make everything dreamy" → sets all windows to 60% opacity
- "Focus mode" → hides all but current window
- "Cyberpunk vibes" → purple borders, high transparency
- "Clean slate" → balanced windows, no overlaps
- "Reference mode" → current window 100%, others 40%

Small model (like Llama 3.2 1B) could run locally and translate vibes into Yabai commands.

## 🌌 Understanding Spaces
Spaces = Virtual Desktops, but Yabai makes them powerful:
- Each space can have different layouts (Space 1: tiled, Space 2: floating)
- Windows belong to specific spaces
- You can name them ("coding", "research", "chill")
- Think of them as different rooms in your digital house
- Yabai can move windows between spaces programmatically

Quick mental model: Monitors > Spaces > Windows

## 🚫 The Tab Problem
**Observation:** Yabai treats tabbed windows weirdly - gives blank space for hidden tabs, window jumps around when switching tabs.

**Solution:** Don't use tabs! 
- Multiple Terminal windows > Terminal tabs
- Multiple Finder windows > Finder tabs

**Philosophy:** Tabs hide context. Multiple windows show everything. Maybe tabs are productivity theater - feeling organized while actually adding friction?

When everything is visible and positioned, you don't need to remember what's in which tab. Your spatial memory kicks in instead.
